Niveau 2 - Controle et collections

(System) Tu ne manipules plus une valeur seule. Tu prends des decisions et tu parcours des series de donnees.

[Critical Node] Le couple condition + boucle. C'est lui qui transforme une liste brute en resultat utile.

[Blueprint]

collection -> lecture -> test -> action -> resultat

Lis le schema comme une chaine de tri sur tapis roulant: chaque element passe devant une porte de controle.

Ce que tu dois maitriser

  • if / elif / else
  • for et while
  • in et not in
  • indexation [] et slicing [:]
  • list, tuple, dict, set

[Mechanics]

notes = [12, 7, 18, 10]
total = 0

for note in notes:
    if note >= 10:
        total += note

print(total)

Ici, for fait avancer le tapis. if decide si la piece entre dans la caisse finale.

Operateurs a verrouiller

  • in : tester la presence d'une valeur
  • not in : tester l'absence
  • [] : acceder a un element
  • [:] : extraire une tranche
  • += : accumuler proprement
  • | & - sur les sets : union, intersection, difference

[Line Audit]

  • notes = [...] : Assignment. Creation de collection.
  • total = 0 : Assignment. Accumulateur.
  • for note in notes : Loop control. Parcours d'une liste.
  • if note >= 10 : Condition + Comparison. Filtre.
  • total += note : Arithmetic + Assignment. Addition accumulee. Inside loop.
  • print(total) : Call. Affichage final. Outside loop.
