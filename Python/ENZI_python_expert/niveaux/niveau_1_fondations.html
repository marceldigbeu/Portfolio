Niveau 1 - Fondations

(System) Ici tu apprends a manipuler une seule valeur correctement. Tant que ce niveau n'est pas stable, tout le reste parait flou.

[Critical Node] Savoir quel type tu manipules. Le bon operateur depend toujours du type.

[Blueprint]

valeur -> type -> operateur -> resultat

Lis cette chaine comme un petit circuit electrique: si tu te trompes de type, le courant logique part mal.

Ce que tu dois maitriser

  • affectation: =
  • calculs: + - * / // % **
  • comparaisons: == != < <= > >=
  • logique: and or not
  • conversion: int() float() str() bool()

[Pseudo-code]

prendre une valeur
identifier son type
appliquer le bon operateur
verifier le resultat affiche

[Mechanics]

x = 10
y = 3

print(x + y)   # 13
print(x / y)   # division reelle
print(x // y)  # division entiere
print(x % y)   # reste
print(x ** y)  # puissance
print(x > y)   # comparaison
print(x > 0 and y > 0)  # logique

Analogie machine: le type est la piece, l'operateur est l'outil. Un tournevis sur un boulon, ca glisse. Un mauvais operateur sur le mauvais type fait pareil.

[Line Audit]

  • x = 10 : Assignment. Tu stockes une valeur entiere.
  • y = 3 : Assignment. Deuxieme valeur entiere.
  • x + y : Arithmetic. Addition.
  • x / y : Arithmetic. Division float.
  • x // y : Arithmetic. Division entiere.
  • x % y : Arithmetic. Reste de division.
  • x ** y : Arithmetic. Puissance.
  • x > y : Comparison. Retourne un bool.
  • x > 0 and y > 0 : Comparison + Condition logic.

Mini guide operateurs du niveau 1

  • = : ranger une valeur dans une variable
  • + : additionner des nombres ou concatener des chaines
  • // : utile quand tu veux un quotient entier
  • % : indispensable pour parite, cycles, decoupages
  • == : comparer des valeurs
  • and : demander si deux conditions sont vraies
