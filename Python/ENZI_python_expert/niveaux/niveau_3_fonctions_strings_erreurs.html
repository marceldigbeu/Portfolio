Niveau 3 - Fonctions, strings, erreurs

(System) Tu passes du script local a un code reutilisable. Tu organises des blocs, tu renvoies des valeurs, tu traites du texte, tu controles les cas qui cassent.

[Critical Node] return, pas print. Une fonction doit produire une valeur, pas juste du bruit console.

[Blueprint]

entree -> fonction -> traitement -> return -> reutilisation

Lis le schema comme une usine: la fonction est une machine. Si elle ne renvoie rien, la ligne de production ne peut pas continuer.

[Pseudo-code]

recevoir des parametres
verifier les cas limites
transformer les donnees
return le resultat

[Mechanics]

def nettoyer_nom(nom: str) -> str:
    if not nom:
        return ""
    return nom.strip().title()

texte = nettoyer_nom("  ada lovelace ")
print(texte)

Operateurs et outils a verrouiller

  • not pour tester le vide logique
  • + sur les strings pour concatener, mais avec mesure
  • in sur une chaine pour chercher un motif
  • is et is not pour None
  • try / except pour contenir une erreur de conversion

[Line Audit]

  • def nettoyer_nom(...) : Definition. Creation d'une fonction.
  • if not nom : Condition. Test de chaine vide.
  • return "" : Return. Cas limite.
  • nom.strip().title() : Call chain. Nettoyage puis mise en forme.
  • texte = nettoyer_nom(...) : Assignment + Call.
  • print(texte) : Affichage dans le programme principal.
